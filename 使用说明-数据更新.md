# 数据更新使用说明 · How to update data

「一人上传，所有人看」的每周流程。只需要会**拖文件**，不碰任何控制台。

---

## 每周更新数据（只有你 A 需要做 · GitHub 自动部署版）

1. **双击打开** `data-injector.html`（数据注入工具，在你电脑上本地打开即可，全程不上传任何数据）。
2. 把本周的销售 `.xlsx` 文件**拖进去**（或点击选择）。工具会显示数据行数、门店数、日期范围确认无误。
3. 点 **下载 index.html** —— 得到一个已经含本周数据的 `index.html`（通常在「下载」文件夹）。
4. 把下载的 `index.html` **移动到项目文件夹** `Desktop/MINISO Dashboard`，**覆盖**原来的同名文件。
5. 打开 **GitHub Desktop** → 左下角填一句说明（如 `更新本周数据`）→ 点 **Commit to main** → 点 **Push origin**。
6. Vercel 会自动检测到更新并重新部署，约 1 分钟后线上即为本周数据。

完成 ✅ 同事打开站点刷新即可看到最新数据。

## 其他同事（什么都不用做）

打开 `miniso-neon.vercel.app` → 直接看到本周最新数据，无需上传、无需任何操作。

---

## 常见问题

- **数据安全吗？** 解析在你本机浏览器完成，注入工具不联网上传数据；数据只是被「烤进」HTML 文件里。
- **表头要求？** 和你一直用的销售底表一致即可（需包含 门店、日期、销售金额、客单量、销售数量、面积、城市、开业时间 等列，中英文表头都支持）。
- **同事打开看到的是旧数据？** 让他们刷新一次页面；部署后约 1 分钟内全网生效。

## 给维护者（技术备注）

- `retail_dashboard.html` 是看板本体（空白/上传版）。其中有一个 `#bakedData` 数据槽，默认 `null` —— 此时行为和原来完全一样（走上传 / IndexedDB 缓存）。
- `data-injector.html` 由 `python3 build-injector.py` 从当前的 `retail_dashboard.html` 生成（把看板模板 base64 嵌入注入工具）。
- **改动看板后，务必重新运行** `python3 build-injector.py`，让注入工具用上最新模板。
- 注入工具把原始 `sheet_to_json` 行直接烤进数据槽，看板加载时用**同一个** `ingestRows()` 处理 —— 与手动上传的结果完全一致，零逻辑重复。
